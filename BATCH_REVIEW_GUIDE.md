# 批次審核功能使用指南

**版本**: 1.0.0
**更新日期**: 2025-12-19

---

## 🎯 功能概述

批次審核功能允許您一次性審核多筆記錄,大幅提升審核效率。透過預設評分模板,您可以在 2-3 分鐘內完成原本需要 30-50 分鐘的工作。

**適用場景**:
- 同類型產品的批次處理
- 相同來源的記錄審核
- 品質明確一致的記錄群組

---

## 🚀 快速開始

### 步驟 1: 選擇記錄

1. 前往 **審核佇列** 頁面 (`/review/queue`)
2. (可選) 使用篩選功能縮小範圍
   - 驗證狀態: PASS/WARN/FAIL
   - 信心水平: HIGH/MEDIUM/LOW
3. 勾選要批次處理的記錄:
   - 單擊 Checkbox 選擇個別記錄
   - 點擊表頭 Checkbox 全選當前頁

**提示**: 建議選擇特徵相似的記錄進行批次處理

### 步驟 2: 開啟批次審核

1. 選擇完成後,頂部會出現工具列
2. 工具列顯示: "已選擇 N 筆記錄"
3. 點擊 **"批次審核"** 按鈕
4. 批次審核 Modal 會自動開啟

### 步驟 3: 選擇評分模板

批次審核 Modal 提供 5 個預設模板:

| 模板 | 品質分數 | 信心分數 | 黃金樣本 | 適用場景 |
|-----|---------|---------|---------|---------|
| ⭐ 優質樣本 | 9 | 0.95 | ✅ 是 | 高品質,適合訓練 |
| ✓ 良好品質 | 8 | 0.85 | ❌ 否 | 品質良好 |
| ○ 可接受 | 7 | 0.75 | ❌ 否 | 品質尚可 |
| △ 需改進 | 5 | 0.6 | ❌ 否 | 品質較差 |
| ✕ 低品質 | 3 | 0.4 | ❌ 否 | 品質不佳 |

**選擇方式**: 點擊任一模板卡片,評分會自動填入

### 步驟 4: 確認或調整評分

1. 檢查自動填入的評分是否合適
2. 如需調整:
   - 品質分數: 1-10 整數
   - 信心分數: 0-1 (0-100%)
3. (可選) 輸入批次審核備註
4. (可選) 勾選/取消 "標記為黃金樣本"

### 步驟 5: 提交批次審核

1. 檢查確認提示框的資訊
2. 點擊 **"批次提交 N 筆"** 按鈕
3. 系統開始處理:
   - 顯示提交進度
   - "處理中..." 狀態
4. 完成後:
   - 顯示成功 Toast 通知
   - Modal 自動關閉
   - 佇列自動更新

## 📊 批次審核 Modal 詳解

### 記錄摘要區塊

顯示選擇記錄的統計資訊:
- **總計**: 選擇的記錄總數
- **FAIL**: 驗證失敗的記錄數
- **WARN**: 驗證警告的記錄數
- **LOW 信心**: 低信心度的記錄數

**警告提示**: 如果批次中包含 FAIL 或 LOW 信心記錄,會顯示黃色警告框,建議謹慎評分。

### 模板選擇區塊

5 個預設模板以卡片形式呈現:
- **圖示 + 名稱**: 快速識別
- **描述**: 適用場景說明
- **評分預覽**: 顯示品質/信心/黃金標記
- **選中狀態**: 藍色邊框高亮

### 評分調整區塊

兩個評分輸入框:
- **品質分數**: 數字輸入,旁邊顯示大字體當前值
- **信心分數**: 數字輸入,旁邊顯示百分比

即時驗證錯誤會以紅色文字顯示。

### 備註輸入區塊

- 文字區域可輸入批次備註
- 備註會套用到所有選擇的記錄
- 提示: "此備註將套用到所有 N 筆記錄"

### 黃金樣本選項

- Checkbox 控制是否批次標記為黃金樣本
- 勾選後顯示: "⭐ 將標記 N 筆"
- 注意: 黃金樣本需符合驗證規則

### 確認提示框

藍色提示框總結批次操作:
- 將對 **N 筆記錄** 套用...
- 品質分數 **X**
- 信心分數 **X%**
- (如有) 並標記為黃金樣本

## 💡 使用技巧

### 技巧 1: 分批處理策略

**建議批次大小**:
- 小批次 (5-10筆): 快速驗證,靈活調整
- 中批次 (10-20筆): 平衡效率與風險
- 大批次 (20+筆): 最高效率,但需確保品質一致

**分批策略**:
```
情境: 50 筆 PASS 狀態記錄需要審核

方案 A (保守):
- 第 1 批: 選擇 10 筆,用 "良好品質" 模板
- 第 2 批: 選擇 15 筆,用 "良好品質" 模板
- 第 3 批: 選擇 25 筆,用 "良好品質" 模板

方案 B (高效):
- 全選 50 筆,統一用 "良好品質" 模板
- 適用於已確認品質一致的情況
```

### 技巧 2: 模板快速選擇

**記憶模板圖示**:
- ⭐ = 優質 (9分)
- ✓ = 良好 (8分)
- ○ = 可接受 (7分)
- △ = 需改進 (5分)
- ✕ = 低品質 (3分)

**快速判斷**:
1. PASS + HIGH 信心 → 優質或良好
2. PASS + MEDIUM 信心 → 良好或可接受
3. WARN → 可接受或需改進
4. FAIL → 需改進或低品質

### 技巧 3: 善用篩選功能

**高效工作流程**:
```
1. 篩選 "PASS" 狀態
2. 批次處理為 "良好品質"
3. 清除篩選

4. 篩選 "WARN" 狀態
5. 批次處理為 "可接受"
6. 清除篩選

7. 篩選 "FAIL" 狀態
8. 逐筆審核或小批次處理
```

### 技巧 4: 備註標準化

建立批次備註模板:
- "批次驗證通過 - 產品資訊完整"
- "批次處理 - 來源: XX平台,品質良好"
- "批次複查 - 修正後確認無誤"
- "批次標記 - 高品質訓練樣本"

### 技巧 5: 黃金樣本篩選

**黃金樣本標準**:
- 品質分數 ≥ 8
- 信心分數 ≥ 0.85
- 備註至少 10 個字
- 資料完整且正確

**批次標記流程**:
1. 篩選 PASS + HIGH 信心
2. 進入批次審核
3. 選擇 "優質樣本" 模板
4. 確認自動勾選黃金樣本
5. 輸入詳細備註 (如: "高品質完整資料,適合訓練模型")
6. 批次提交

## ⚠️ 注意事項

### 1. 品質一致性

批次處理前,確保選擇的記錄具有相似特徵:
- ✅ 相同來源
- ✅ 相似品質水平
- ✅ 一致的驗證狀態
- ❌ 不要混合 PASS 和 FAIL
- ❌ 不要混合不同品質等級

### 2. 驗證規則

即使是批次操作,系統仍會執行相同的驗證規則:
- 品質分數必須是 1-10 整數
- 信心分數必須是 0-1
- 黃金樣本需符合特定條件
- 高品質應搭配高信心度

**違反規則時**: 系統會阻止提交並顯示錯誤訊息

### 3. 提交時間

批次提交採用逐筆處理:
- 5 筆記錄約需 2-5 秒
- 10 筆記錄約需 5-10 秒
- 20 筆記錄約需 10-20 秒

**提交期間**:
- 按鈕顯示 "處理中..."
- 無法關閉 Modal
- 請耐心等待

### 4. 失敗處理

如果部分記錄提交失敗:
- Toast 通知會顯示成功/失敗數量
- 例如: "批次審核完成:成功 18 筆,失敗 2 筆"
- 失敗的記錄會保留在佇列中
- 建議稍後重新審核失敗記錄

### 5. 不可撤銷

批次審核提交後:
- ❌ 無法撤銷
- ✅ 記錄會從佇列中移除
- ✅ 建立 Ground Truth 記錄
- ✅ 更新統計資料

**建議**: 提交前仔細檢查確認提示框

## ❓ 常見問題

### Q1: 可以批次處理不同驗證狀態的記錄嗎?

A: 技術上可以,但 **不建議**。不同驗證狀態通常代表不同的品質水平,應該分別處理以確保評分準確性。

### Q2: 批次提交失敗了怎麼辦?

A:
1. 查看 Toast 通知的錯誤訊息
2. 失敗的記錄仍會保留在佇列中
3. 可以重新選擇失敗的記錄再次提交
4. 如持續失敗,聯繫技術支援

### Q3: 如何知道哪些模板最適合?

A: 參考模板描述和以下指引:
- PASS + HIGH → 優質或良好
- PASS + MEDIUM → 良好或可接受
- WARN → 可接受或需改進
- FAIL → 需改進或低品質

### Q4: 批次審核會影響統計資料嗎?

A: 會的,批次提交的記錄會:
- 從審核佇列中移除
- 計入已審核總數
- 更新品質統計
- 影響儀表板數據

### Q5: 可以中途取消批次提交嗎?

A: 提交開始後 **無法取消**。建議:
- 提交前仔細檢查
- 使用小批次測試
- 確認無誤後再大批次處理
